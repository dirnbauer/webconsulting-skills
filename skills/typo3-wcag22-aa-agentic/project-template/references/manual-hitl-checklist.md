# Minimal HITL WCAG 2.2 AA checklist

Use this after automated fixes and before publishing the Erklärung zur Barrierefreiheit.

## Keyboard

- [ ] Tab order follows visible/logical order.
- [ ] Skip link appears and works.
- [ ] Main navigation can be opened/closed by keyboard.
- [ ] Mobile navigation can be opened/closed by keyboard.
- [ ] Dropdowns/accordions/modals do not trap focus.
- [ ] Escape closes overlays where expected.
- [ ] Focus returns to opener after modal/menu close.
- [ ] Visible focus indicator is clear on all controls.

## Screenreader smoke test

- [ ] Page title is meaningful.
- [ ] H1 and heading order make sense.
- [ ] Landmarks are available and not duplicated incorrectly.
- [ ] Navigation current page is announced.
- [ ] Buttons/links have meaningful names.
- [ ] Form fields, errors and required fields are announced.
- [ ] Dynamic status messages are announced when relevant.

## Content/editorial

- [ ] Informative images have meaningful alt text.
- [ ] Decorative images have empty alt text.
- [ ] Link texts are understandable.
- [ ] PDFs are accessible or listed as exceptions/alternatives.
- [ ] Videos have captions/transcripts where in scope.
- [ ] Maps/third-party embeds have accessible alternatives.

## WCAG 2.2 specific review

- [ ] Focus not obscured by sticky headers/cookie bars.
- [ ] Dragging actions have non-drag alternatives.
- [ ] Target sizes are acceptable or exceptions documented.
- [ ] Login/authentication does not require inaccessible cognitive puzzles.
- [ ] Repeated entry is avoided where applicable.

## Statement/legal

- [ ] Correct legal profile selected: WZG, Landes-WZG, BaFG or voluntary.
- [ ] Status wording approved: fully, partially, or not compliant/vereinbar.
- [ ] Non-accessible content described in user language.
- [ ] Workarounds/alternatives included where relevant.
- [ ] Contact and enforcement procedure correct.
- [ ] Annual review reminder/ticket created.
